# Lab Manual: Mock Help Desk & Support
Version: 1.0
Author: Pradip Pandey
Difficulty: Beginner
Estimated Time: 1 Hour

## 1. Project Overview
This lab simulates an L1/L2 Support environment. We will set up a ticketing system, simulate common user issues, and document resolutions in a Knowledge Base (KB).

## 2. Lab Requirements
*   **Ticketing System**: Spiceworks Cloud (Free) or Freshservice (Trial).
*   **Test VMs**: One "Admin" PC, one "Broken" User PC.

## 3. Step-by-Step Implementation

### Phase 1: Ticketing System Setup
1.  **Register**: Sign up for Spiceworks/Freshservice.
2.  **Configure Portal**:
    *   Set Company Name: "Bright Rays IT".
    *   Create Categories: Hardware, Software, Network.
    *   Define SLA (Service Level Agreement):
        *   Critical: 1 Hour Response.
        *   Standard: 4 Hour Response.

### Phase 2: Simulating Incidents
We will purposely break the "User PC" to solve tickets.

#### Scenario A: "Printer Offline"
1.  **Break**: Stop the "Print Spooler" service (`net stop spooler`).
2.  **Report**: User logs ticket "Cannot Print".
3.  **Resolve**:
    *   Connect remotely.
    *   Command: `net start spooler`.
    *   Verify printer status.
    *   Close ticket with notes.

#### Scenario B: "Internet Sluggish"
1.  **Break**: Change DNS to an invalid server (e.g., 10.0.0.99).
2.  **Diagnose**:
    *   `ping google.com` (Fails).
    *   `ping 8.8.8.8` (Success). **Diagnosis: DNS Issue**.
3.  **Resolve**:
    *   Set DNS to `8.8.8.8` or DHCP.
    *   Update Ticket.

### Phase 3: Knowledge Base (KB)
Prevent future tickets by documenting solutions.

1.  **Write KB Article**:
    *   **Title**: How to Map a Network Drive.
    *   **Step 1**: Open File Explorer.
    *   **Step 2**: Right-click "This PC" -> Map Network Drive.
    *   **Step 3**: Enter path `\\server\share`.
    *   **Step 4**: Enter credentials.

## 4. Conclusion
Help Desk isn't just resetting passwords. It requires logical troubleshooting (OSI Model), patience, and clear documentation.
