# Lab Manual: Digital Forensics Investigation
Version: 1.0
Author: Pradip Pandey
Difficulty: Advanced
Estimated Time: 2 Hours

## 1. Project Overview
This scenario places you as a Forensic Analyst investigating a USB drive seized from a suspect accused of leaking application source code. The goal is to create a forensic image, recover deleted files, and find evidence of web activity.

## 2. Lab Requirements
*   Software: Autopsy (Open Source), FTK Imager.
*   Hardware: A USB Flash Drive (Target).

## 3. Step-by-Step Implementation

### Phase 1: Evidence Acquisition (FTK Imager)
*Never analyze the original drive directly. Always analyze a copy (image).*

1.  Insert Suspect Drive: Plug in the USB.
2.  Create Disk Image:
    *   Open FTK Imager (as Admin).
    *   File -> Create Disk Image.
    *   Source: Physical Drive (Select USB).
    *   Destination: Add -> E01 (EnCase format). This is industry standard as it supports compression and metadata.
    *   Case Info: Enter Case ID `CF-001`.
    *   Image Fragment Size: Default (`1500MB`).
3.  Verification:
    *   Once finished, FTK will calculate MD5/SHA1 Hashes.
    *   Record these. They are the digital fingerprint of the potential evidence.

### Phase 2: Analysis (Autopsy)
1.  Create Case:
    *   Open Autopsy. New Case -> `Case Name: Leak_Investigation`.
2.  Add Data Source:
    *   Select Disk Image or VM file.
    *   Browse to the `.E01` file you created.
3.  Configure Ingest Modules:
    *   Select: Recent Activity, File Type Identification, Deleted Files, Keyword Search.
    *   Unselect time-intensive modules like "Encryption Detection" for this small lab.
4.  Run Ingest: This will process the image (indexing files, extracting Registry data).

### Phase 3: Finding Evidence
1.  Recovering Deleted Files:
    *   Navigate the tree to Deleted Files (FileSystem).
    *   Look for documents (PDF, DOCX).
    *   Right-click -> Export to recover a file the suspect thought was gone.
2.  Web Artifacts:
    *   Go to Web History.
    *   Search for "Dropbox" or "Google Drive" to see if files were uploaded to the cloud.
3.  Keyword Search:
    *   Search for "Confidential" or "Source Code".
    *   Autopsy will highlight every occurence on the disk (even in unallocated space).

### Phase 4: Registry Analysis
1.  Persistence:
    *   Check `Software\Microsoft\Windows\CurrentVersion\Run`.
    *   Malware often adds keys here to start automatically on boot.

## 4. Reporting
1.  Generate Report:
    *   Click Generate Report.
    *   Format: HTML or PDF.
    *   Select the specific tagged files (the deleted PDF, the web history).
2.  Chain of Custody: Ensure your report references the original Hash values to prove data integrity.

## 5. Conclusion
Digital Forensics requires strict adherence to procedures. By imaging the drive first and verifying hashes, we ensured the evidence is admissible in court.
