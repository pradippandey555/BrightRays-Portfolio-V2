# Lab Manual: Windows Server Active Directory
**Version:** 1.0
**Author:** Pradip Pandey
**Difficulty:** Intermediate
**Estimated Time:** 2 Hours

## 1. Project Overview
This lab walks through setting up a Windows Domain Environment from scratch. You will promote a server to a Domain Controller (DC), create Users and Groups, and apply Group Policy Objects (GPOs) to enforce security settings on client computers.

## 2. Lab Requirements
*   **Server VM**: Windows Server 2019 or 2022.
*   **Client VM**: Windows 10 Enterprise.
*   **Network**: Both VMs on same Internal Network (Static IP for Server).

## 3. Step-by-Step Implementation

### Phase 1: Initial Configuration
1.  **Set Static IP**:
    *   Server Address: `192.168.1.10`
    *   Subnet: `255.255.255.0`
    *   Gateway: `192.168.1.1`
    *   DNS: `127.0.0.1` (Self)
2.  **Rename Computer**:
    *   Rename to `DC01`. Restart.

### Phase 2: Install AD DS Role
1.  Open **Server Manager**.
2.  Manage -> **Add Roles and Features**.
3.  Select **Active Directory Domain Services**. Include Management Tools.
4.  Proceed and Install.

### Phase 3: Promote to Domain Controller
1.  Click the Warning Flag in Server Manager -> "Promote this server to a domain controller".
2.  **Deployment Operation**: Select **Add a new forest**.
3.  **Root Domain Name**: `corp.local`.
4.  **DSRM Password**: Set a complex password.
5.  Paths: Leave default (`C:\Windows\NTDS`).
6.  Install and Reboot.

### Phase 4: User & Group Management
1.  Open **Active Directory Users and Computers** (ADUC).
2.  **Create OUs**:
    *   Right Create Domain -> New -> Organizational Unit.
    *   Name: `Departments`, `IT_Admins`.
3.  **Create User**:
    *   In `Departments`, New -> User.
    *   Name: `Jane Doe`. Logon: `j.doe`. Password: Complex.
4.  **Create Group**:
    *   New -> Group -> `Files_ReadWrite`.
    *   Add Jane Doe to this group.

### Phase 5: Group Policy (GPO)
We will create a specific policy to disable USB drives.

1.  Open **Group Policy Management**.
2.  Expand Forest -> Domains -> corp.local.
3.  Right-click `Departments` OU -> "Create a GPO in this domain, and Link it here".
4.  Name: `Secure_Workstations`.
5.  **Edit Policy**:
    *   Right-click `Secure_Workstations` -> Edit.
    *   Navigate: **Computer Configuration** -> **Policies** -> **Admin Templates** -> **System** -> **Removable Storage Access**.
    *   Find: **All Removable Storage classes: Deny all access**.
    *   Set to: **Enabled**.
6.  **Update Policy**:
    *   Run `gpupdate /force` on a client machine.

## 4. Verification
1.  **Join Client**: Set Windows 10 DNS to `192.168.1.10`. System Properties -> Join Domain -> `corp.local`.
2.  **Login**: Log in as `CORP\j.doe`.
3.  **Test GPO**: Plug in a USB drive. Access should be Denied.

## 5. Conclusion
Active Directory is the heart of Enterprise IT. You have successfully deployed a Domain, organized users, and centrally managed security settings via Group Policy.
