# Lab Manual: Linux Server Hardening
Version: 1.0
Author: Pradip Pandey
Difficulty: Intermediate
Estimated Time: 1 Hour

## 1. Project Overview
Linux servers are the backbone of the internet. In this lab, setting up an Ubuntu Server is just step one. The real task is **Hardening** it—locking it down to CIS Benchmark standards to resist attacks.

## 2. Lab Requirements
*   **Virtual Machine**: Ubuntu Server 20.04 or 22.04 LTS.

## 3. Step-by-Step Implementation

### Phase 1: User & Access Management
1.  **Create Admin User**:
    *   Never log in as `root`.
    *   `sudo adduser admin`
    *   `sudo usermod -aG sudo admin`
2.  **Lock Root Account**:
    *   `sudo passwd -l root`

### Phase 2: SSH Hardening
This is the most critical step. Botnets scan port 22 24/7.

1.  **Edit Config**:
    `sudo nano /etc/ssh/sshd_config`
2.  **Key Changes**:
    *   **Port**: Change from `22` to `2222`. (Security by Obscurity, but reduces noise).
    *   **PermitRootLogin**: Change to `no`.
    *   **PasswordAuthentication**: Change to `no` (Must setup SSH Keys first!).
    *   **MaxAuthTries**: Set to `3`.
3.  **Restart Service**:
    `sudo systemctl restart sshd`

### Phase 3: Firewall (UFW)
Ubuntu comes with Uncomplicated Firewall (UFW).

1.  **Default Policy**:
    *   Deny all incoming, allow all outgoing.
    *   `sudo ufw default deny incoming`
    *   `sudo ufw default allow outgoing`
2.  **Allow Rules**:
    *   **SSH (Custom Port)**: `sudo ufw allow 2222/tcp`
    *   **Web**: `sudo ufw allow 80/tcp`
    *   **SSL**: `sudo ufw allow 443/tcp`
3.  **Enable**:
    *   `sudo ufw enable`
    *   *Warning*: Ensure you allowed SSH first, or you will lock yourself out!

### Phase 4: Intrusion Prevention (Fail2Ban)
Fail2Ban scans log files and bans IPs that show malicious signs.

1.  **Install**:
    `sudo apt install fail2ban`
2.  **Configure Jail**:
    *   Create local config: `sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local`
    *   Edit: `sudo nano /etc/fail2ban/jail.local`
    *   Find `[sshd]`. Set `enabled = true`. Set `port = 2222`.
3.  **Restart**:
    `sudo systemctl restart fail2ban`
4.  **Verify**:
    `sudo fail2ban-client status sshd`

### Phase 5: Automatic Updates
Security patches must be applied immediately.
1.  **Install**:
    `sudo apt install unattended-upgrades`
2.  **Configure**:
    `sudo dpkg-reconfigure --priority=low unattended-upgrades`

## 4. Verification Check
Running a hardening script or audit tool like **Lynis**.
1.  `sudo apt install lynis`
2.  `sudo lynis audit system`
3.  Check your Hardening Index Score (Aim for > 75).

## 5. Conclusion
A default Linux install is not secure. By disabling root, using firewalls, and implementing Fail2Ban, we significantly reduced the attack surface of this server.
