# Lab Manual: Enterprise Switching & Routing (Cisco)
**Version:** 2.0
**Author:** Pradip Pandey
**Difficulty:** Intermediate
**Estimated Time:** 1.5 Hours

## 1. Project Overview
This lab simulates a small enterprise network using Cisco Packet Tracer (or physical hardware). We will configure Layer 2 Switching (VLANs, Security) and Layer 3 Routing (Inter-VLAN, OSPF).

## 2. Lab Requirements
*   **Packet Tracer Student** or **GNS3**.
*   **Hardware (Optional)**: 1x Cisco 2960 Switch, 1x Cisco 1941 Router.

## 3. Network Topology
*   **Router (R1)**: Gateway for the network.
*   **Switch (SW1)**: Connected to R1 via G0/1.
*   **PC1 (HR Dept)**: VLAN 10.
*   **PC2 (IT Dept)**: VLAN 20.

## 4. Step-by-Step Implementation

### Phase 1: Layer 2 Switching (VLANs)
1.  **Create VLANs**:
    ```cisco
    SW1(config)# vlan 10
    SW1(config-vlan)# name DATA_HR
    SW1(config)# vlan 20
    SW1(config-vlan)# name DATA_IT
    ```
2.  **Assign Access Ports**:
    *   PC1 connected to Fa0/1.
    ```cisco
    SW1(config)# interface fa0/1
    SW1(config-if)# switchport mode access
    SW1(config-if)# switchport access vlan 10
    ```
    *   PC2 connected to Fa0/2.
    ```cisco
    SW1(config)# interface fa0/2
    SW1(config-if)# switchport mode access
    SW1(config-if)# switchport access vlan 20
    ```

### Phase 2: Router-on-a-Stick (Inter-VLAN Routing)
Since a Layer 2 switch cannot route packets between VLAN 10 and 20, we need the Router.

1.  **Configure Trunk Link**:
    *   Connection from SW1 (Trunk) to R1.
    ```cisco
    SW1(config)# interface g0/1
    SW1(config-if)# switchport mode trunk
    ```
2.  **Configure Sub-Interfaces on Router**:
    *   For VLAN 10:
    ```cisco
    R1(config)# interface g0/0.10
    R1(config-subif)# encapsulation dot1q 10
    R1(config-subif)# ip address 192.168.10.1 255.255.255.0
    ```
    *   For VLAN 20:
    ```cisco
    R1(config)# interface g0/0.20
    R1(config-subif)# encapsulation dot1q 20
    R1(config-subif)# ip address 192.168.20.1 255.255.255.0
    ```
    *   No Shutdown Physical Interface:
    ```cisco
    R1(config)# interface g0/0
    R1(config-if)# no shutdown
    ```

### Phase 3: Dynamic Routing (OSPF)
If we added another router (R2), we would need a routing protocol.

1.  **Configure OSPF ID 1**:
    ```cisco
    R1(config)# router ospf 1
    R1(config-router)# network 192.168.10.0 0.0.0.255 area 0
    R1(config-router)# network 192.168.20.0 0.0.0.255 area 0
    ```
2.  **Security (Passive Interface)**:
    Do not send OSPF Hello packets to PC networks.
    ```cisco
    R1(config-router)# passive-interface g0/0.10
    R1(config-router)# passive-interface g0/0.20
    ```

### Phase 4: Port Security
Secure the access ports to prevent unauthorized devices.

1.  **Enable Port Security**:
    ```cisco
    SW1(config)# interface fa0/1
    SW1(config-if)# switchport port-security
    SW1(config-if)# switchport port-security maximum 1
    SW1(config-if)# switchport port-security mac-address sticky
    SW1(config-if)# switchport port-security violation shutdown
    ```
    *Explanation*: Only 1 MAC address allowed (learned dynamically). If a second device plugs in, the port shuts down.

## 5. Verification
1.  **Ping Test**:
    *   PC1 (192.168.10.10) ping PC2 (192.168.20.10).
    *   *Result*: Successful (Packets go PC1 -> SW1 -> R1 -> SW1 -> PC2).
2.  **Check Routes**:
    *   `R1# show ip route`

## 6. Conclusion
This lab establishes a fundamental segmented network. Best practices used include Port Security (Access Control) and Passive Interfaces (Routing Security).
