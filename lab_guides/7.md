# Lab Manual: IT Project Management Simulation
**Version:** 1.0
**Author:** Pradip Pandey
**Difficulty:** Beginner (Administrative)
**Estimated Time:** 2 Hours

## 1. Projects Overview
This lab simulates the documentation and workflow required to manage a large-scale IT project: "Migrating On-Premise Servers to Cloud".

## 2. Lab Requirements
*   **Software**: Microsoft Project (or ProjectLibre), Excel, Word.

## 3. Step-by-Step Implementation

### Phase 1: Initiation
1.  **Create Project Charter (Word)**:
    *   **Project Title**: Cloud Migration 2024.
    *   **Business Case**: Current hardware is end-of-life (EOL). Moving to Azure reduces CapEx.
    *   **Scope**: Migrate 50 Virtual Machines.
    *   **Budget**: $50,000 USD.
    *   **Sponsor**: CIO.

### Phase 2: Planning
1.  **Work Breakdown Structure (WBS)**:
    *   **1.0 Assessment**: Audit Apps, Dependency Mapping.
    *   **2.0 Design**: Network Arch, Security.
    *   **3.0 Pilot**: Migrate 5 non-critical VMs.
    *   **4.0 Full Migration**: Weekend Cutover.
2.  **Gantt Chart (ProjectLibre)**:
    *   Input WBS tasks.
    *   Set Dependencies (Design must finish before Pilot starts).
    *   Assign Resources (Cloud Engineer, Net Admin).
3.  **Risk Register (Excel)**:
    *   **Risk**: "Unexpected Bandwidth Bottleneck".
    *   **Probability**: High. **Impact**: High.
    *   **Mitigation**: Upgrade ExpressRoute link before migration.

### Phase 3: Execution (Agile)
1.  **Kanban Board (Trello)**:
    *   Setup columns: **Backlog**, **In Progress**, **Testing**, **Done**.
    *   Create Cards: "Config VNET", "Sync AD", "Test Latency".
    *   Move cards during simulated Daily Standups.
2.  **Status Reporting**:
    *   Weekly "Red/Amber/Green" status emails to Stakeholders.

### Phase 4: Closing
1.  **Lessons Learned Meeting**:
    *   What went well? (Pilot revealed a firewall issue early).
    *   What went wrong? (DNS propagation took longer than expected).
2.  **Project Sign-off**: Formal acceptance from Sponsor.

## 4. Conclusion
Effective PM methodologies (Waterfall for planning, Agile for execution) ensure IT projects are delivered on time and under budget.
