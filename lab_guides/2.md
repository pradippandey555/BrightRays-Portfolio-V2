# Lab Manual: Cloud Infrastructure (Azure/AWS)
**Version:** 1.0
**Author:** Pradip Pandey
**Difficulty:** Advanced
**Estimated Time:** 3 Hours

## 1. Project Overview
This project builds a Hybrid Cloud Infrastructure simulation. We will provision a Virtual Network in Azure, deploy a Web Server, and secure it using Network Security Groups (NSGs). We will also simulate a "Hybrid" connection as if connecting to an on-premise office.

## 2. Lab Requirements & Setup
*   **Microsoft Azure Account** (Student or Free Tier).
*   **Azure CLI** installed on your desktop (`winget install -e --id Microsoft.AzureCLI`).
*   **Visual Studio Code** for editing Terraform/Scripts.

## 3. Step-by-Step Implementation

### Phase 1: Resource Group & VNET (Azure Portal)
1.  **Login**: Go to [portal.azure.com](https://portal.azure.com).
2.  **Create Resource Group**:
    *   Search "Resource groups" -> Create.
    *   Name: `rg-hybrid-lab`.
    *   Region: `East US` (or closest).
3.  **Create Virtual Network (VNet)**:
    *   Search "Virtual networks" -> Create.
    *   Name: `vnet-cloud-prod`.
    *   Address Space: `10.1.0.0/16`.
    *   **Subnets**:
        *   `subnet-frontend` (10.1.1.0/24) - For Web Servers.
        *   `subnet-backend` (10.1.2.0/24) - For Databases.

### Phase 2: Security Configuration (NSG)
We operate on the principle of "Zero Trust". We explicitly allow only needed traffic.

1.  **Create Network Security Group**:
    *   Search "Network security groups" -> Create `nsg-frontend`.
2.  **Inbound Security Rules**:
    *   **Rule 1 (HTTP)**:
        *   Source: Any
        *   Destination Port: 80
        *   Action: Allow
        *   Priority: 100
        *   Name: `Allow_HTTP`
    *   **Rule 2 (SSH)**:
        *   Source: IP Addresses (Enter your home Public IP).
        *   *Why?* Do not open port 22 to the world!
        *   Destination Port: 22
        *   Action: Allow
        *   Priority: 110
        *   Name: `Allow_SSH_Mgmt`
3.  **Associate NSG**:
    *   Go to Subnets -> Associate `nsg-frontend` with `subnet-frontend`.

### Phase 3: Compute Deployment (VM)
1.  **Create Virtual Machine**:
    *   Search "Virtual machines" -> Create.
    *   Name: `vm-web-01`.
    *   Image: **Ubuntu Server 20.04 LTS**.
    *   Size: **Standard_B1s** (Student/Free eligible).
    *   Authentication: **SSH Public Key**.
    *   Networking: Select `vnet-cloud-prod` and `subnet-frontend`.
    *   Public IP: Create new.
2.  **Deployment**: Click Review + Create. Download the `.pem` key file (KEEP THIS SAFE).

### Phase 4: Web Server Configuration (Linux)
1.  **Connect via SSH**:
    *   Open Terminal (or CMD/PowerShell).
    *   `ssh -i <path_to_key.pem> azureuser@<VM_Public_IP>`
2.  **Install Nginx**:
    ```bash
    sudo apt update
    sudo apt install nginx -y
    ```
3.  **Verify**:
    *   Open your browser and type the VM's Public IP.
    *   You should see "Welcome to nginx!".

### Phase 5: Infrastructure as Code (Terraform) - Optional
To automate this in the future, use Terraform.
1.  Create `main.tf`.
2.  Define Provider:
    ```hcl
    provider "azurerm" {
      features {}
    }
    ```
3.  Define Resources (Group, VNet, Subnet).
4.  Run `terraform init` -> `terraform apply`.

## 4. Troubleshooting
*   **SSH Timeout**: Check your specific NSG rule. Your Home IP might have changed. Update the source IP in the NSG rule.
*   **Site not loading**: Check standard Ubuntu firewall (`sudo ufw status`). It might need `sudo ufw allow 'Nginx HTTP'`.

## 5. Conclusion
You have successfully deployed a secure cloud infrastructure. This demonstrates core Cloud Engineer skills: Virtual Networking, Compute Management, and Network Security.
